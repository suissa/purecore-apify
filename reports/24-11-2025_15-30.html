✨ Funcionalidades Implementadas

🔄 Circuit Breaker

Proteção automática contra falhas em cascata com threshold configurável de 5 falhas e reset automático após 10 segundos.

failureThreshold: 5
resetTimeoutMs: 10000

⏱️ Timeout Inteligente

Timeout de 30 segundos com sistema de retry automático, chunking de requests grandes e canal WS para processamento paralelo.

timeoutMs: 30000
maxTimeoutMs: 60000
retryAttempts: 3

🔗 WS Retry Channel

Canal WebSocket dedicado para processamento de retries em paralelo, com filas de processamento e telemetria completa.

enableAsyncChannel: true
enableTelemetry: true
enableRequestChunking: true

📊 Observabilidade

Logs estruturados, métricas de performance e traces distribuídos para monitoramento completo da aplicação.

@Logs() @Metrics() @TraceSpan()

🔐 Segurança JWT + NO_AUTH

Autenticação JWT obrigatória com sistema NO_AUTH para rotas públicas configuráveis via ambiente.

NO_AUTH="GET /health, POST /login"

🛡️ XSS Protection

Proteção automática contra ataques XSS em todos os inputs (body, query, params) exceto rotas públicas.

@XSSGuard - sanitização automática

🚀 Smart Cache

Cache inteligente com TTL de 5 minutos para otimização de performance e redução de carga.

ttl: 300 segundos
@SmartCache automático

🔄 Sistema NO_AUTH

Configuração via .env para excluir rotas específicas da autenticação JWT.

NO_AUTH="{method} /{entity}, ..."

📝 Como Usar

1. Configuração do Ambiente

# Copie o template cp src/env-config.ts .env # Configure as variáveis JWT_SECRET=your-super-secret-jwt-key-change-this-in-production NO_AUTH="GET /health, POST /login, GET /status"

2. Uso Simples nos Controllers

import { ApifyCompleteSentinel } from '@purecore-br/4pi'; class UsersController { @ApifyCompleteSentinel async list(req, res) { // Tudo já está configurado! res.json({ users: [] }); } }

3. Excluindo Rotas da Autenticação

# .env NO_AUTH="GET /health, POST /login, GET /api/v1/public/info, PUT /users/profile"
