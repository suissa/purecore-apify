# 🚀 Exemplo AutoGeneratorDDD

Este diretório contém exemplos completos de como usar o **AutoGeneratorDDD** para gerar arquitetura **Domain-Driven Design (DDD)** baseada em schemas Zod.

## 📁 Arquivos

- **`user.schema.ts`** - Schema Zod completo para entidade User
- **`test-ddd-generator.ts`** - Exemplos de uso do AutoGeneratorDDD
- **`README-DDD.md`** - Esta documentação

## 🏗️ O que o DDD Generator Cria

Para cada entidade definida em um schema Zod, o generator cria **15+ arquivos** organizados em **4 camadas DDD**:

### 1. Domain Layer (🏛️ Camada de Domínio)
```
domain/
├── entities/user.entity.ts          # Entidade pura de domínio
├── value-objects/email.vo.ts        # Value Objects (Email, Password, etc.)
├── aggregates/userAggregate.ts      # Aggregate Root
├── events/user.events.ts            # Domain Events
├── services/user.domain-service.ts  # Domain Services
└── repositories/iuser.repository.ts # Repository Interface
```

### 2. Application Layer (📱 Camada de Aplicação)
```
application/
├── commands/user.commands.ts        # CQRS Commands
├── queries/user.queries.ts          # CQRS Queries
├── handlers/
│   ├── user.command-handlers.ts     # Command Handlers
│   └── user.query-handlers.ts       # Query Handlers
├── dtos/user.dto.ts                 # Data Transfer Objects
└── services/user.app-service.ts     # Application Service
```

### 3. Infrastructure Layer (🔧 Camada de Infraestrutura)
```
infrastructure/
├── repositories/user.repository.ts   # Repository Implementation
├── database/
│   ├── user.schema.ts               # Database Schema
│   └── context.ts                   # Database Context
├── external-services/
│   └── user.external-service.ts     # External API Integration
└── config/database.config.ts        # Infrastructure Config
```

### 4. Presentation Layer (🌐 Camada de Apresentação)
```
presentation/
├── controllers/user.controller.ts    # REST Controller
├── routes/user.routes.ts            # Route Definitions
└── middlewares/user.middleware.ts   # Custom Middlewares
```

### Cross-Cutting Concerns (🎯 Preocupações Transversais)
```
tests/user.test.ts                    # Comprehensive Tests
shared/constants.ts                   # Shared Constants
index.ts                             # Module Exports
```

## 🚀 Como Testar

### 1. Executar Exemplo Simples

```bash
# No diretório raiz do projeto
bun run examples/test-ddd-generator.ts
```

### 2. Gerar Arquitetura Real (não dry-run)

Edite o arquivo `examples/test-ddd-generator.ts` e mude `dryRun: true` para `dryRun: false`:

```typescript
await autoGenerateDDDFromZodSchemas({
  modulesPath: 'src/modules',
  domainPath: 'src/domain',
  boundedContext: 'user-management',
  verbose: true,
  force: false,
  dryRun: false // <-- Mude para false
});
```

## 🎯 Funcionalidades Demonstradas

### CQRS Pattern
```typescript
// Commands (escrita)
CreateUserCommand, UpdateUserCommand, DeleteUserCommand

// Queries (leitura)
GetUserByIdQuery, GetAllUsersQuery, SearchUsersQuery

// Handlers separados
UserCommandHandlers, UserQueryHandlers
```

### Domain-Driven Design
```typescript
// Entidade de domínio pura
export class User {
  // Regra de negócio na entidade
  public validate(): boolean {
    // Validações de domínio
    return true;
  }
}

// Aggregate Root
export class UserAggregate {
  private root: User;
  private domainEvents: DomainEvent[] = [];

  performBusinessOperation() {
    // Lógica de negócio do agregado
    this.addDomainEvent(new UserBusinessOperationPerformedEvent(this.root.id));
  }
}
```

### Clean Architecture
- **Domain Layer**: 100% independente de frameworks
- **Application Layer**: Coordena domínio e infraestrutura
- **Infrastructure Layer**: Detalhes técnicos isolados
- **Presentation Layer**: Interface externa

## ⚙️ Configurações

### Opções do Generator

```typescript
interface DDDGenerationOptions {
  modulesPath?: string;      // Caminho dos módulos (padrão: 'src/modules')
  domainPath?: string;       // Caminho do domínio (padrão: 'src/domain')
  force?: boolean;           // Forçar regeração (padrão: false)
  verbose?: boolean;         // Log detalhado (padrão: true)
  dryRun?: boolean;          // Apenas simular (padrão: false)
  boundedContext?: string;   // Contexto delimitado (padrão: 'default')
}
```

### Variáveis de Ambiente

```bash
# Configurações de banco
DB_HOST=localhost
DB_PORT=5432
DB_NAME=app_db
DB_USER=user

EXTERNAL_TIMEOUT=5000

# JWT
JWT_SECRET=your-jwt-secret
```
